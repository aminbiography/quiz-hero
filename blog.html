  • Question 1: Question 1: What is the difference in terms of map, filter, foreach and find in JavaScript?
  • Answer Q.1:
    map: It creates a new array with the results of calling a provided function on every element in the array.
    Such as:
    const numbers = [1, 2, 3, 4, 5];
    const doubledNumbers = numbers.map(number => number * 2);
    console.log(doubledNumbers);
    // Output: [2, 4, 6, 8, 10]

    filter: It creates a new array with all elements that pass the test implemented by the provided function.
    Such as:
    const numbers = [1, 2, 3, 4, 5];
    const evenNumbers = numbers.filter(number => number % 2 === 0);
    console.log(evenNumbers);
    // Output: [2, 4]

    forEach: It executes a provided function once for each element in an array.
    Such as:
    const numbers = [1, 2, 3, 4, 5];
    numbers.forEach(number => console.log(number));
    // Output:
    // 1
    // 2
    // 3
    // 4
    // 5

    find: It returns the first element in an array that satisfies a provided testing function.
    Such as:
    const numbers = [1, 3, 5, 2, 4, 6];
    const firstEvenNumber = numbers.find(number => number % 2 === 0);
    console.log(firstEvenNumber);
    // Output: 2
  • Question 2: What is the difference in terms of local storage session storage and cookies in JavaScript?
  • Answer Q.2:
    Local storage and session storage store data on the client side and have different lifetimes, while cookies can be used for server-side data management and have expiration dates.
    For example, local storage can be used to persist data across browser sessions, session storage can be used to store data for a specific session, and cookies can be used to store user preferences or login information.
  • Question 3: What is an event loop in javascript and how does it work?
  • Answer Q.3:
    The event loop is a key part of the JavaScript runtime environment that allows for the handling of asynchronous events. It constantly checks if there are any pending tasks in the task queue and executes them when the call stack is empty. This allows for non-blocking operations to be performed efficiently, preventing the browser from freezing.
    For example, when using the setTimeout function, the event loop adds the task to the task queue with a delay specified by the function, and when the delay is up, the function is added to the call stack and executed. The event loop then repeats this process, checking for new tasks in the task queue and executing them when the call stack is empty.
  • Question 4:The quizTimer Function: how does it work?
  • Answer Q.4:
    The quizTimer function starts a setInterval loop that executes every second. Inside the loop, it calculates the elapsed time in minutes and seconds, formats the display with leading zeros, and updates an HTML element with the timer value. After every 60 seconds, it creates a flashing effect by changing the text color of the timer. The function can be dismissed by passing a parameter that clears the setInterval loop. This function is useful for creating timed quizzes or countdowns where users need to complete a task within a set time limit.
